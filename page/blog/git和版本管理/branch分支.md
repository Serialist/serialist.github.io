# branch（分支）

一个仓库可以有多个分支，这用于分离代码，如多人合作，防止污染原来代码等等。

branch 是工程文件的单元，如我们常在创建一个主分支叫做 main 或 master，再创建一个 dev（development）分支，在 dev 分支中工作，最终把做好的合并到主分支。

在你本地的仓库也有分支，他镜像于你的远程分支（当然也可以到别的仓库上）。你可以把本地分支的 commit push 到你的远程仓库中。

合并就是把一个分支并入另一个分支，即 pr（pull request，这是在 GitHub 的常用说法。在 GitLab 也叫做 merge）。两个分支可能有不同的地方，即冲突。通过取舍有冲突的代码后，可以完成 merge，完结你在子分支的工作。字如其名，pull request，他的过程和你 pull 云端代码发现和本地代码有冲突时的流程差不多。

当你在 fork 了另一个人的仓库，就在本地创建了一个类似于他的仓库的分支，你可以在 fork 到自己手底下的仓库的分支中工作，然后给那个人发 pr。这时候 pull request 的 request 就很清楚了：你自己仓库分支的 pr 是给自己发请求，而 fork 时是 request 别人来 pull 你的分支。这是最常见的，给其他项目做贡献的工作方式。
