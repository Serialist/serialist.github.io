# 版本号

版本号一般形如：`v1.2.3.20260101_alpha`，它包括：

- 前缀 vversion 版本的缩写。

- 主版本号（v**2**.1.3.20250412_alpha）

  当功能模块有较大的变动，比如增加模块或是整体架构发生变化。此版本号由项目经理决定是否修改。

- 副版本号（v2.**1**.3.20250412_alpha）

  当功能有一定的增加或变化，比如增加了对权限控制、增加自定义视图等功能。此版本号由项目经理决定是否修改。

- 修订版本号（v2.1.**3**.20250412_alpha）

  般是 bug 修复或是一些小的变动，要经常发布修订版，时间间隔不限，修复一个严重的 bug 即可发布一个修订版。此版本号由项目经理决定是否修改。

- 日期版本号（v2.1.3.**20250412**_alpha）

  用于记录修改项目的当前日期，每天对项目的修改都需要更改日期版本号。此版本号由开发人员决定是否修改。

- 希腊字母版本号（v2.1.3.20250412_**alpha**）

  希腊字母版本号包括很多，常见的有：base、alpha、beta 、RC 、 release。

  - base

    基础框架版本。通常包括所有功能，但没有完整实现，只是做为整体的基础架构。

  - alpha

    内测版本，初级版本。此阶段以实现功能为主，通常只在开发者内部交流。该版本软件 bug 一般较多，是测试版。

    测试人员提交 bug 经开发人员修改，发布给测试人员，此时可标注 alpha 版。

  - beta

    公测版本。该版本消除了严重错误，但还需要多次测试来进一步消除。

    修改 bug 经测试人员确认发布到公网上，此时可标注 beta 版。

  - RC

    RC 是 release candidate 的缩写，是“发布候选版”。该版本已经相当成熟，基本不存在导致错误的 bug，与发行的正式版本相差无几。

  - release

    发行版，是“最终版本”。在一系列测试后，终归有一个正式版本，是最终交付用户的版本。有时也称标准版。
